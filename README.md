# homelab-testing-lab
Overview<br>
<br>
<br>
This project is a personal homelab environment built on Unraid and containerized services to simulate real-world system behavior and reliability scenarios.
<br>
<br>
The goal of this project is not just media streaming, but system and software testing practice, including service validation, failure scenario testing, and basic automation.
<br>
<br>
This environment is used to practice concepts similar to software test engineering, including:<br>
<br>
System reliability testing<br>
Service health validation<br>
Failure recovery scenarios<br>
Basic API and automation testing<br>
System Architecture<br>
<br>
The environment includes containerized services running via Docker, such as:<br>
<br>
Media streaming service (Plex)<br>
Automation services (Sonarr / Radarr)<br>
File storage and management services<br>
Testing Objectives<br>
<br>
Planned enhancements include:<br>
<br>
Expanding automated test coverage with pytest<br>
Adding API testing for service endpoints<br>
Logging and reporting test results<br>
Simulating more complex failure scenarios<br>
Integrating CI-style automated test runs<br>
Key Learning Outcomes<br>
<br>
This project helps develop skills in:<br>
<br>
System testing and validation thinking<br>
Basic test automation with Python<br>
Understanding service reliability and failure modes<br>
Structuring test cases and documentation<br>
<br>
Notes<br>
<br>
This project is part of a broader transition into software and system test engineering, building on experience in regulated medical device environments under frameworks such as ISO 13485.
