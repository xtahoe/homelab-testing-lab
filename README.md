# homelab-testing-lab
Overview

This project is a personal homelab environment built on Unraid and containerized services to simulate real-world system behavior and reliability scenarios.

The goal of this project is not just media streaming, but system and software testing practice, including service validation, failure scenario testing, and basic automation.

This environment is used to practice concepts similar to software test engineering, including:

System reliability testing
Service health validation
Failure recovery scenarios
Basic API and automation testing
System Architecture

The environment includes containerized services running via Docker, such as:

Media streaming service (Plex)
Automation services (Sonarr / Radarr)
File storage and management services
Testing Objectives

Planned enhancements include:

Expanding automated test coverage with pytest
Adding API testing for service endpoints
Logging and reporting test results
Simulating more complex failure scenarios
Integrating CI-style automated test runs
Key Learning Outcomes

This project helps develop skills in:

System testing and validation thinking
Basic test automation with Python
Understanding service reliability and failure modes
Structuring test cases and documentation
Notes

This project is part of a broader transition into software and system test engineering, building on experience in regulated medical device environments under frameworks such as ISO 13485.
